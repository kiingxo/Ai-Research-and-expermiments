Revisiting Quantization

Quantization is the process of converting a model from a higher-precision format to a lower-precision format.
This is done to reduce memory usage and increase inference speed — with only a slight sacrifice in model performance or accuracy.

For example, let’s assume we have a large language model (LLM) originally trained in full precision (32-bit float). During quantization, this model can be converted to 16-bit (FP16) or even 8-bit (INT8) formats, significantly reducing the size and compute requirements.


Why Quantization Matters During Fine-tuning

When fine-tuning LLMs (e.g., LLaMA, Mistral, or MPT), especially on consumer-grade hardware (like a laptop with 16GB RAM or a 24GB GPU), full-precision models may be too large to fit in memory.

🔧 Example Use Case:

You’re fine-tuning a 7B parameter LLaMA model:
	•	The FP32 version may require over 60–80GB of VRAM.
	•	The 4-bit quantized version (e.g., QLoRA) can run and be fine-tuned on just 16–24GB VRAM.

This makes it possible to:
	•	Run the model on a single GPU (e.g., RTX 3090 or Apple M2 with 32GB RAM)
	•	Use techniques like QLoRA (Quantized Low-Rank Adaptation) to fine-tune efficiently without sacrificing much accuracy
