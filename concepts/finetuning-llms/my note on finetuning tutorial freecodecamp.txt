Full Precision vs Half Precision
	•	Full precision: Typically uses 32-bit floating point (FP32)
	•	Half precision: Uses 16-bit (FP16 or BF16), reducing memory and compute usage

⸻

Revisiting Quantization

Quantization is the process of converting a model from a higher-precision format to a lower-precision format.

Goal:
	•	Reduce memory footprint
	•	Speed up inference
	•	Keep model accuracy close to original

Example:
	•	Original model trained in FP32 (32-bit float)
	•	Can be quantized to:
	•	FP16 (16-bit float)
	•	INT8 (8-bit integer)
	•	Even 4-bit (used in QLoRA)

⸻

Why Quantization Matters During Fine-tuning

When fine-tuning LLMs (like LLaMA, Mistral, or MPT) on laptops or consumer GPUs, full-precision models may not fit in memory.

🔧 Example Use Case:
Fine-tuning a 7B LLaMA model:
	•	FP32 version needs ~60–80GB VRAM
	•	4-bit quantized version (QLoRA) fits into 16–24GB VRAM

This makes it possible to:
	•	Run on one GPU (e.g., RTX 3090 or Apple M2 with 32–64GB RAM)
	•	Use QLoRA for fine-tuning with low memory but good accuracy

⸻

Types of Quantization
	1.	Post-Training Quantization: Applied after model training.
	2.	Quantization-Aware Training (QAT): Model is trained with quantization in mind.

Precision Levels
	•	FP32: Full precision (baseline)
	•	FP16 / BF16: Half precision
	•	INT8: 8-bit integer quantization
	•	4-bit: Aggressive compression (used in QLoRA)

⸻

Symmetric vs Asymmetric Quantization

Symmetric Quantization:
	•	Zero-point is fixed at 0
	•	Same scale factor is used across the tensor
	•	More efficient, but less flexible

Asymmetric Quantization:
	•	Non-zero zero-point
	•	Each tensor (or channel) can have different scale/zero-point
	•	More accurate, especially for weights or activations not centered around 0
